## 一、已完成的功能实现

### 1.1 后端约束逻辑
**文件位置**: `Dormitory_business/src/main/java/com/example/springboot/service/impl/StudentServiceImpl.java:72-90`

**实现内容**:
- 在 `updateNewStudent()` 方法中添加了寝室长约束检查
- 当设置学生为寝室长时，先检查该宿舍是否已有寝室长
- 如果已有寝室长，抛出异常并提示"该宿舍已有寝室长：XXX"
- 排除当前正在编辑的学生（避免自己与自己冲突）

**核心代码**:
```java
if (student.getIsRoomLeader() != null && student.getIsRoomLeader() == 1) {
    if (student.getDormRoomId() != null) {
        QueryWrapper<Student> qw = new QueryWrapper<>();
        qw.eq("dorm_room_id", student.getDormRoomId());
        qw.eq("is_room_leader", 1);
        if (student.getUsername() != null) {
            qw.ne("username", student.getUsername());
        }
        Student existingRoomLeader = studentMapper.selectOne(qw);
        if (existingRoomLeader != null) {
            throw new RuntimeException("该宿舍已有寝室长：" + existingRoomLeader.getName());
        }
    }
}
```

## 三、系统架构说明

### 数据模型
**Student 实体类字段**:
- `isRoomLeader` (Integer): 寝室长标识，0=否，1=是

### 数据库字段
- 表名: `student`
- 字段名: `is_room_leader`
- 类型: INT
- 默认值: 0

### 权限控制流程
```
宿舍管理员登录
    ↓
进入学生信息页面
    ↓
选择学生 → 编辑 → 设置寝室长
    ↓
后端检查该宿舍是否已有寝室长
    ↓
如果有 → 报错提示
如果没有 → 保存成功

学生登录
    ↓
判断 isRoomLeader 字段
    ↓
是寝室长 → 显示标识，可以申请报修
不是寝室长 → 不显示标识，只能查看
```

## 五、注意事项

1. **学生必须有宿舍才能设置为寝室长**
   - 如果学生没有分配宿舍（dormRoomId 为空），无法设置为寝室长

2. **修改寝室长需要先取消**
   - 如果要更换寝室长，需要先将当前寝室长取消，再设置新的

3. **前端会根据用户身份显示不同内容**
   - 宿舍管理员：可以看到和编辑"是否寝室长"字段
   - 系统管理员：看不到"是否寝室长"字段
   - 学生：在"我的宿舍"页面看到寝室长标识

4. **报修申请的权限判断**
   - 权限判断基于登录用户的 `isRoomLeader` 字段
   - 该字段在登录时存入 sessionStorage
